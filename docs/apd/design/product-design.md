# AWS EC2 GPU Quick Reference — Product Design

## 1. Who（誰のため）

- **AWS Solutions Architect (SA)**: お客様へのGPUインスタンス提案時に、スペック・価格・リージョン情報を素早く参照したい
- **AWS Sales**: 商談中にGPUインスタンスの概要を即座に見せ、お客様との議論に使いたい
- **AWSでGPUを利用したいお客様**: ML/AIワークロードに最適なインスタンスを、スペックと価格の両面から自分で比較検討したい

**共通の課題**: GPUインスタンスの情報がAWS公式ドキュメント、価格ページ、ブログ記事などに散在しており、横断的な比較に時間がかかる

## 2. Why（なぜ今）

- GPU需要がAI/MLの急成長に伴い拡大し続けている
- AWSのGPUインスタンスは世代・バリエーションが増加（Volta〜Blackwell）し、全体像の把握が困難になっている
- 既存の代替手段の限界:
  - **AWS公式ドキュメント**: インスタンスタイプ別にページが分かれ、GPU世代横断の比較ができない
  - **AWS料金ページ**: スペック情報と価格が別々のページにあり、突き合わせが手間
  - **instances.vantage.sh**: 全EC2インスタンスが対象でGPU特化ではなく、FP16/FP8性能やEFAバージョンなどの情報が不足
  - **各種ブログ記事**: 執筆時点の情報で古くなりがち

## 3. What（何ができるか）

- 全世代のAWS GPU（NVIDIA）インスタンスを**1つのテーブルで一覧比較**できる
- GPUスペック（VRAM、FP16/FP8性能）、インスタンススペック（vCPU、メモリ、NVMe）、ネットワーク（EFA、PCIe）を一目で確認
- **On-Demand価格とCapacity Blocks価格**を並べて比較できる
- **東京リージョンの利用可否**がひと目でわかる
- **簡易コスト計算ツール**で、利用時間・GPU数に応じた概算コストを算出できる
- **多言語対応**（日本語・英語）で、海外のお客様やグローバルチームとも共有できる
- **洗練されたデザイン**で、商談やプレゼンの場でも見せられるクオリティ

## 4. What Not（何をやらないか）

1. **NVIDIA以外のアクセラレータ**（AWS Trainium/Inferentia等）は現時点では対象外。スペック体系が異なるため、将来的に別テーブルとしての追加を検討
2. **リアルタイム価格連動**はしない（手動更新、または定期バッチ更新）
3. **インスタンスの起動・プロビジョニング機能**は提供しない（あくまで情報参照ツール）
4. **詳細なベンチマーク結果**の掲載はしない（公式スペック値のみ）
5. **ユーザーアカウント・ログイン機能**は不要（静的サイトのまま維持）
6. **モバイルアプリ化**はしない（レスポンシブWebで対応）
7. **全リージョンの価格比較**はスコープ外（東京リージョンを軸に、On-Demand基準価格を掲載）

## 5. FAQ

### ステークホルダー視点

1. **Q: なぜ静的サイトなのか？**
   A: GitHub Pagesで無料ホスティングでき、CDN配信で高速。バックエンド不要でセキュリティリスクも低い。ビルドツールの導入は許容するが、最終出力は静的サイトとする

2. **Q: データの更新頻度は？**
   A: AWS価格やインスタンスの変更に合わせて随時更新。目標は新インスタンス発表後1週間以内

3. **Q: 多言語対応の範囲は？**
   A: まず日本語・英語の2言語。UI文言とカラム名を切り替え、データ自体は言語非依存

### ユーザー視点

4. **Q: 価格情報は正確か？**
   A: AWS Price List APIおよびCapacity Blocks価格データを情報源とし、更新日を明示する。ただし最終確認はAWS公式を推奨

5. **Q: 自社のワークロードに最適なインスタンスをどう選べばいいか？**
   A: テーブルで各インスタンスのスペックを比較し、コスト計算ツールで概算コストを確認できる

6. **Q: モバイルでも使えるか？**
   A: レスポンシブ対応済み。ただしテーブルの特性上、PC/タブレットでの閲覧を推奨

7. **Q: オフラインで使えるか？**
   A: 静的HTMLのため、ページを保存すればオフラインでも参照可能

### エンジニア視点

8. **Q: なぜ単一HTMLファイル構成なのか？**
   A: 依存関係ゼロでポータビリティが高い。共有・保存が容易で、社内展開のハードルが低い

9. **Q: コスト計算ツールの精度は？**
   A: 単純な時間×単価の概算。RI/Savings Plans等の複雑な割引は対象外

10. **Q: 新しいGPUインスタンスの追加はどうするか？**
    A: `GPU_DATA`配列にエントリを追加するだけ。データ駆動設計のため、HTMLテンプレートの変更は不要

## 6. Success Criteria

- AWS SA/Salesが商談中に実際にリファレンスとして使用している
- お客様からのフィードバックでインスタンス選定の意思決定が迅速化したとの声
- 社内外のSNS・ブログで有用なツールとして紹介される
