# D-001: ビルドツール導入によるファイル分割

## コンテキスト

現在のプロダクトは全て `index.html` 1ファイルに含まれている（CSS、HTML、JS、データ）。
多言語対応・コスト計算ツール・ライト/ダークモード追加により、コード量が大幅に増加する見込み（1700行→3000行超）。
Design文書にて「ビルドツールの導入は許容するが、最終出力は静的サイトとする」と方針決定済み。

## 選択肢

1. **単一HTMLを維持** — 現状通り全てを1ファイルに含める
2. **ビルドツール導入で分割** — 開発時はファイル分割し、ビルドで結合
3. **複数ファイル構成** — CSS/JS/データを分離し、個別にロード

## 判断

**選択肢2: ビルドツール導入で分割**

## 理由

- 開発時はCSS / JS / 翻訳データ / GPU_DATAを個別ファイルに分離し、メンテ性を確保
- ビルド時に全てインラインで結合し、最終出力は単一HTMLを維持
- Design文書の方針と整合
- 単一HTML維持により、ポータビリティ（保存・共有の容易さ）も損なわない
